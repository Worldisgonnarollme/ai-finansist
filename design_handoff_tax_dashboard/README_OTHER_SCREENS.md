# Handoff: остальные экраны приложения (не только Дашборд)

Дополнение к основному `README.md` (там — только экран «Главная»/Дашборд
вариант 1). Здесь — Dart-код для всех остальных экранов из HTML-прототипа
`Tax App Prototype.dc.html`, в той же light-палитре (белый/бежевый/зелёный/
рыжий), теми же токенами (`lib/core/theme/*`).

## Fidelity
High-fidelity — цвета, отступы, радиусы и копирайт финальные.

## Файлы

### Виджеты (`lib/widgets/`)
- `segmented_toggle.dart` — переключатель Самозанятый/ИП (track-style) и
  Доход/Расход (pill-style, Add Transaction).
- `regime_option_tile.dart` — карточка выбора налогового режима (радио +
  название + описание + ставка), плюс `RegimeFamilyLabel` (заголовок
  группы "УСН"/"Патент"/"ОСНО"/"НПД").
- `app_bottom_nav_bar.dart` — нижняя навигация, 4 таба, **подписи видны**
  (в отличие от старого приложения, где подписи были скрыты — по брифу
  "чтобы всё было видно").
- `period_card.dart` — карточка месяца в «Истории» (доход/налог/взносы +
  прогресс-бар доли налога).
- `statement_card.dart` — карточка загруженной выписки во «Выписках».
- `settings_section.dart` — `SettingsSection` (заголовок + карточка) и
  `SettingsRow` (строка с чевроном/детейлом/Switch) для Настроек,
  Налогового режима, Банков.
- `bank_tile.dart` — строка банка (подключённого — со статусом и
  "Отключить"; или доступного для подключения — с чевроном).
- `screen_header.dart` — переиспользуемый заголовок вложенного экрана:
  `BackButtonPill` (кнопка "назад") + `ScreenHeader` (шапка целиком).

### Экраны (`lib/screens/`)
- `onboarding_screen.dart` — 3 слайда, "Пропустить", точки пагинации,
  "Далее"/"Начать".
- `tax_mode_screen.dart` — выбор статуса + режима (первый запуск).
- `history_screen.dart` — вкладка «История»: список `PeriodCard`.
- `period_detail_screen.dart` — детали месяца: сводка + фильтр-чипы + список
  операций.
- `statements_screen.dart` — вкладка «Выписки».
- `settings_screen.dart` — вкладка «Настройки».
- `profile_screen.dart` — Профиль (Имя/Телефон/ИНН/ОГРНИП + Сохранить).
- `tax_regime_screen.dart` — обзор текущего режима + карточка оптимизации.
- `tax_regime_select_screen.dart` — смена режима (тот же список, что и на
  Tax Mode).
- `tax_regime_details_screen.dart` — настройки режима: сотрудники, ставка
  НДС, освобождение от НДС.
- `connected_banks_screen.dart` — список подключённых банков + "Добавить
  банк".
- `bank_selection_screen.dart` — список банков для подключения + баннер
  безопасности.
- `bank_consent_screen.dart` — согласие на передачу данных конкретного
  банка.
- `bank_loading_screen.dart` — 3-шаговая анимация подключения → галочка →
  автовозврат.
- `add_transaction_screen.dart` — добавление операции: тип, сумма, категория,
  дата, описание.
- `main_shell.dart` — иллюстративная сборка: 4 таба + push-навигация по
  всем вложенным экранам через `Navigator`. Замените на роутинг вашего
  проекта (`onGenerateRoute` и т.п., как в текущем `main.dart`) — здесь
  показана только схема связей между экранами.

## Копирайт и цифры
Все тексты и цифры (суммы, даты, проценты, названия банков) — из
прототипа, для демонстрации макета на реалистичных данных. Замените на
реальные данные пользователя/расчёта при интеграции — **не меняйте
формулы и бизнес-правила**, которые уже реализованы в проекте.

## Что нужно донастроить при интеграции
- Подключить реальные данные вместо статичных списков (`_periods`,
  `_statements`, `_allTx`, `_banks` и т.д.) — они сейчас захардкожены как
  в HTML-прототипе.
- `TaxRegimeDetailsScreen` показан для ОСНО (сотрудники + ставка НДС +
  освобождение). Для УСН/Патента/НПД нужен свой набор полей — переиспользуйте
  тот же визуальный каркас (`ScreenHeader` + секции + кнопка "Сохранить").
- Валидацию форм (Профиль, Новая операция) нужно добавить — в макете поля
  либо статичны, либо без валидации.
- `main_shell.dart` — не production-роутинг, а иллюстрация связей между
  экранами; интегрируйте в существующую систему роутов проекта.
