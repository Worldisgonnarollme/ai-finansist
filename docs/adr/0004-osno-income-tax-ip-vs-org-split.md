# ADR-0004: `tax.osno.income_tax` — раздельные правила для ИП (НДФЛ) и организации (налог на прибыль)

## Статус
Принято

## Контекст
Единственное правило `tax.osno.income_tax` цитировало одновременно
ст. 210 НК РФ (НДФЛ, применяется к ИП) и ст. 247 НК РФ (налог на прибыль
организаций, применяется к ООО) для любого дохода, не проверяя
`input.user.is_ip`. Это смешивало две разные по субъекту правовые нормы в
одном основании — фактически правило никогда не было верным целиком: для
ИП неприменима ст. 247, для организации неприменима ст. 210.

## Решение
Правило разделено на два, каждое со своим условием по `input.user.is_ip` и
своим `legal_basis`:
- `is_ip == true` → `legal_basis: ["НК РФ ст. 210"]`.
- `is_ip == false` → `legal_basis: ["НК РФ ст. 247"]`.

Если `is_ip` не передан — нет структурного основания выбрать норму,
`result` падает на anti-hallucination default (`needs_review: true`), а не
угадывает субъект по умолчанию.

В `tax_policy_input.schema.json` поле `user.is_ip` (ранее опциональное с
`default: false`) сделано обязательным условно — только при
`tax_mode == "OSNO"` (через `if`/`then`). Для остальных режимов оно пока
не используется ни в одном правиле и не требуется.

## Отклонённые альтернативы
- **Оставить `default: false` у `is_ip` и считать отсутствие is_ip
  организацией.** Отклонено: это та же «лазейка по умолчанию», которую
  проект уже закрыл для `related_to_business`/`linked_document`/
  `has_vat_invoice` — молчаливое значение по умолчанию для субъекта
  налогообложения подменяет структурный факт угадыванием.
- **Сделать `is_ip` обязательным безусловно (для всех режимов).**
  Отклонено: вне ОСНО `is_ip` сейчас нигде не используется в правилах
  (`tax.policy`), требовать его было бы избыточным расширением модели
  данных без юридического основания для остальных режимов.

## Последствия
- (+) Каждое уверенное (`needs_review: false`) решение об облагаемости
  дохода теперь цитирует ровно ту статью, которая применима к субъекту —
  нет случаев, когда ст. 210 и ст. 247 заявлены одновременно для одной
  операции.
- (-) Любой вызов `tax.osno.income_tax` для ОСНО без `is_ip` в `input`
  теперь не резолвится уверенно (раньше резолвился, но с неверным
  составом legal_basis) — вызывающая сторона обязана знать субъект
  заранее.
