# ADR-0003: `tax.osno.expenses` — явная проверка закрытого перечня ст. 270 НК РФ

## Статус
Принято

## Контекст
`tax_osno_expenses.rego` признавал расход вычитаемым (`deductible: true`)
по ст. 252 НК РФ (обоснованность + документальное подтверждение), но
никогда не проверял ст. 270 НК РФ — закрытый перечень расходов, не
учитываемых при налогообложении НЕЗАВИСИМО от их обоснованности и
документального подтверждения. Это означало, что обоснованный и
документально подтверждённый, но законодательно исключённый расход (например,
прямо попадающий в перечень ст. 270) мог быть ошибочно признан
вычитаемым.

## Решение
В `transaction` добавлено обязательное (для расходных операций) поле
`excluded_under_art_270`. Вызывающая сторона (ИИ или разработчик) обязана
явно подтвердить, входит расход в перечень ст. 270 или нет — без
умолчания.

`tax.osno.expenses` теперь даёт два самостоятельных уверенных ответа:
- `excluded_under_art_270 == true` → `deductible: false`, `needs_review:
  false`, `legal_basis: ["НК РФ ст. 270"]` — это закрытый перечень,
  применяется независимо от ст. 252.
- `excluded_under_art_270 == false` и выполнены условия ст. 252 →
  `deductible: true`, `legal_basis: ["НК РФ ст. 252", "НК РФ ст. 270"]`.

Поле сделано обязательным условно — только когда `direction == "expense"`
(через `if`/`then` в JSON Schema), а не безусловно для всех транзакций,
поскольку для доходных операций вопрос вычитаемости расхода не возникает.

## Отклонённые альтернативы
- **Не добавлять отдельное поле, а описывать исключение текстом в
  `description`.** Отклонено: противоречит принципу проекта — `OPA` не
  угадывает по тексту, оно требует структурный факт.
- **Сделать поле безусловно обязательным для всех транзакций (как
  `related_to_business`/`linked_document`/`has_vat_invoice`).** Отклонено:
  в отличие от тех трёх полей (выбранных пользователем как универсальный
  guard), ст. 270 НК РФ по смыслу применяется только к расходам —
  требовать его для дохода было бы бессмысленным полем без юридического
  основания.

## Последствия
- (+) Закрытый перечень ст. 270 НК РФ больше не может быть случайно
  обойдён через одну только документальную обоснованность по ст. 252.
- (+) Решение об исключении расхода по ст. 270 — это теперь уверенный
  (`needs_review: false`) ответ, а не повод для ручной проверки: закрытый
  перечень не оставляет неоднозначности.
- (-) Любой код, формирующий вход для ОСНО-расходов, должен теперь явно
  знать и передавать `excluded_under_art_270` — это новая обязанность
  вызывающей стороны, ещё не реализованная в `ai_service.dart`/`Transaction`
  (OPA-слой пока не подключён к приложению).
